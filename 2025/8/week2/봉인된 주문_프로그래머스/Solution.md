# 봉인된 주문

## 문제 설명
- 문제 링크: [링크](https://school.programmers.co.kr/learn/courses/30/lessons/389481)
- 핵심 사항:
  - "당신은 삭제가 완료된 주문서에서 n번째 주문을 찾아내야 합니다."
  - 1 ≤ n ≤ 10^15
  - 1 ≤ bans의 길이 ≤ 300,000
---

## 접근 방법
- 주문서 구성
  - a~z 까지 26진수라고 생각할 수 있음.
- 26진수이기 때문에 문자열 <-> 숫자의 변환이 가능함
- n이하의 스킵된 주문서의 개수가 m이라고 할때
  - n+m이 원하는 주문서임.
